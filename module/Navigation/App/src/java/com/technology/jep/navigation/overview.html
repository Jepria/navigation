Описание

Данный модуль содержит функционал модуля Навигации, основное назначение которого - представление в виде древовидного иерархического справочника разделов системы, а также подключаемых внешних систем в виде пунктов меню этого дерева. Примеры структуры и подключения внешних модулей (например, NavigationJro) можно посмотреть в файле: App\src\resources\com\technology\jep\navigation\menu\navigation.xml.
Предыдущая версия модуля, реализованная с использованием библиотеки JepCommon, обладала рядом недостатков, которые были устранены в данной версии:
  • Использование устаревших html-тегов frameset и frame (удалены из HTML5)
  • Отсутствие возможности открытия нескольких модулей на одной странице и быстрого переключения между ними
  • Сложность кастомизации внешнего вида и добавление персональных настроек отдельных пользователей
  • Жесткая зависимость между основным приложением навигации и встраиваемого файла структуры, что приводило к необходимости пересборки модуля для новых добавляемых модулей
и ряд других неудобств, которые были учтены и реализованы в текущей версии модуля, позволяющие cделать работу пользователей с системой гораздо удобнее.
Ниже перечислены основные улучшения модуля
  1. Интегрирование вложенных модулей посредством добавления iframe в DOM-дерево приложения
  2. Вкладочный механизм (возможность одновременной работы с несколькими модулями).
  3. Меню настроек (возможность кастомизации модуля при работе с навигацией – среди прочих возможность отключение вкладочного режима как в старой версии, а также возможность повторно перезагружать содержимое уже открытых вкладок при повторном открытии).
  4. Навигации внешних систем хранятся в отдельных модулях. Данное нововведение позволяет модифицировать дерево меню без пересборки основного приложения Navigation.
  5. Для применения изменений в навигации внеших модулях – теперь достаточно в меню настроек нажать на опцию «Актуализировать меню» (кнопка доступна пользователям с полным доступом прав).
  6. В силу того, что устранено наследие от JepCommon в структурах навигации (xml-файлы) следует избегать использования значений атрибута lastEventInRequest, равное значениям EV_REFRESH, EV_BROWSE и т.д. Вместо этого следует использовать числовые константы – 6, 1 и т.д.

Особенности настройки

В силу того, что в модуле реализована возможность хранения основной навигации в отдельном модуле, важным шагом является указание в переменных окружения ссылки на xml-документ структуры навигации. Для этого необходимо добавить следующие переменные окружения:
  • navigationModuleName со значением, равным контекстному пути модуля, содержащего структуры навигации (например, /NavigationJro).
  • navigationMenuXmlPath со значением, равным относительному пути на файл xml-структуры модуля (например, /WEB-INF/resources/com/technology/jep/navigationjro/menu/navigation.xml)
Также для удобства сброса кэша текущего приложения, у пользователей с полным уровнем доступа добавлена данная функциональность в разделе настроек. По умолчанию, пользователи, обладающие ролью NavAll ({@link com.technology.jep.navigation.navigation.server.NavigationServerConstant#DEFAULT_FULL_ACCESS_ROLE DEFAULT_FULL_ACCESS_ROLE}), имеют доступ к этой функциональности. Если на продукционном окружении значение данной роли отличается от оригинала, то достаточно определить в переменной окружения FULL_ACCESS_ROLE новое наименование роли с полным доступом прав.
Чтобы сохранить привычную ссылку, по которой доступен модуль навигации - необходимо в конфигурационном файле httpd.conf веб-сервера Apache сделать редирект привычной ссылки следующим образом: 
    RewriteRule ^/Navigation/indexInput.do http://<имя_сервера>/Navigation/ [P]

Особенности при работе с Навигацией

Для возможности поддержки персональных настроек пользователей, добавлен раздел Настроек в нижней части левой панели. Среди имеющихся разделов дерева настроек ниже представлено основное назначение каждого из них:
  • Вкладки - включение/отключение режима одновременной работы с несколькими модулями (в выключенном состоянии поведение модуля аналогично предыдущей версии)
  • Актуализировать меню - возможность программной перезагрузки дерева навигации и связанных модулей без перезагрузки контейнера (требуется роль, определенная переменной окружения FULL_ACCESS_ROLE(по умолчанию, NavAll))
  • Обновлять содержимое в повторно открываемой вкладке - во включенном состоянии при повторном клике по меню происходит обновление содержимого вкладки (в выключенном состоянии открытая вкладка станет активной)
  • Сохранять состояние при обновлении страницы - во включенном состоянии при обновлении страницы Навигации будет сохранено текущее рабочее состояние (выделенные пункты меню в дереве Навигации, открытые вкладки)
Данные настройки индивидуальны и сохраняются в cookie браузера на срок 1 год.

Подготовка бинарного репозитория

Для возможности установки данного модуля необходимо сперва развернуть бинарный репозиторий на локальном окружении.
Для этого достаточно скачать модуль Bin в подходящую директорию на локальной файловой системе ОС, ссылка на которую будет использована во время установки модуля.

Установка

Перед установкой необходимо убедиться в наличии бинарного репозитория на локальном окружении, с которого производится установка данного модуля:
  • В случае его отсутствия необходимо выполнить действия из раздела Подготовка бинарного репозитория и продолжить установку.
  • Если бинарный репозиторий уже настроен, то рекомендуется произвести его актуализацию, выполнив команду SVN Update, после чего продолжить установку.
Для продолжения установки текущего модуля, необходимо перейти в директорию App модуля и выполнить команду:
ant deploy -DDEPLOYMENT_PATH=<DEPLOYMENT_PATH> -DLOGIN=<LOGIN> -DPASSWORD=<PASSWORD> -DLOAD_OPERATORID=<LOAD_OPERATORID>  -DBIN_HOME=<BIN_HOME> , где
  • DEPLOYMENT_PATH - адрес установки модуля (Пример для OAS: [opmn://]host:opmnPort[/iASInstanceName/oc4jInstanceName], пример для Tomcat - http[s]://host:port/manager/text)
  • PORT - параметр нужен в случае, если http-порт на сервере отличен от 80го (по-умолчанию значение 80)
  • LOGIN - логин пользователя, под которым происходит установка модуля
  • PASSWORD - пароль пользователя, под которым происходит установка модуля
  • LOAD_OPERATORID - логин/пароль учетной записи в системе, от чьего имени производится установка
  • BIN_HOME - директория бинарного репозитория
Пример: 
ant deploy -DDEPLOYMENT_PATH=opmn://host:opmnPort/OracleAS_1/OC4J_4 -DPORT=8888 -DLOGIN=LOGIN -DPASSWORD=PASSWORD -DLOAD_OPERATORID=user/123 -DBIN_HOME=C:/Project/JEP/Bin
